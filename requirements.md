# STAKEHOLDER
Estudiantes: usuarios principales que solicitan y devuelven bicicletas.
Administración de la universidad: supervisa el funcionamiento del sistema.
Equipo de mantenimiento: este es el grupo que se encarga de reparar bicicletas dañadas.
Seguridad del campus: monitorea el uso adecuado de las bicicletas.
Personal de soporte técnico: mantiene el sistema digital funcionando.

# REQUISITOS FUNCIONALES

1. El sistema debe permitir a los estudiantes consultar las bicicletas disponibles en el campusd
2: El sistema debe permitir a los estudiantes solicitar o reservar una bici
3. El sistema debe registrar la entrega de una bicicleta a un estudiante
4. El sistema debe permitir registrar la devolución de la bici 
5. El sistema debe permitir a los estudiantes reportar problemas o daños en las bicicletas
6. El sistema debe mostrar el historial de préstamos de cada estudiante.

# Requisitos no funcionales

1. El sistema debe estar disponible las 24 horas del día.
2. El sistema debe responder a las solicitudes en menos de 2 segundos.
